com.ibm.juno.server.annotation

Annotation Type Response


  • @Documented
@Target(value=PARAMETER)
@Retention(value=RUNTIME)
@Inherited
public @interface Response
    Annotation used in conjunction with RestMethod.responses() to identify possible responses by the method.
    Example

    @RestMethod( name="*", responses={ @Response(value=200,description="Everything was great."), @Response(value=404,description="File was not found.") @Response(500), } ) public void doAnything(RestRequest req, RestResponse res, @Method String method) { ... }

    Author:
    James Bognar (jbognar@us.ibm.com)

    • Element Detail

      • value

        public abstract int value
        HTTP response code.

      • description

        public abstract String description
        Optional description.

        The default value pulls the description from the description entry in the servlet resource bundle. (e.g. "myMethod.res.[code] = foo" or "MyServlet.myMethod.res.[code] = foo").

        This field can contain variables (e.g. "$L{my.localized.variable}"). See RestServlet.createRequestVarResolver(RestRequest).

        Default:
        ""

      • output

        public abstract Var[] output
        Optional response variables.

        Response variables can also be defined in the servlet resource bundle. (e.g. "myMethod.res.[code].[category].[name] = foo" or "MyServlet.myMethod.res.[code].[category].[name] = foo").

        Default:
        {}
