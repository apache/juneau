352 		 *
353 		 * <p>
354 		 * A comma-delimited list of HTTP method names that are allowed to be passed as values in an <c>X-Method</c> HTTP header
355 		 * to override the real HTTP method name.
356 		 *
357 		 * <p>
358 		 * Allows you to override the actual HTTP method with a simulated method.
359 		 * <br>For example, if an HTTP Client API doesn't support <c>PATCH</c> but does support <c>POST</c> (because

362 		 *
363 		 * <h5 class='section'>Example:</h5>
364 		 * <p class='bjava'>
365 		 * 	<jc>// Option #1 - Defined via annotation resolving to a config file setting with default value.</jc>
366 		 * 	<ja>@Rest</ja>(allowedMethodHeaders=<js>"PATCH"</js>)
367 		 * 	<jk>public class</jk> MyResource {
368 		 *
369 		 * 		<jc>// Option #2 - Defined via builder passed in through resource constructor.</jc>
370 		 * 		<jk>public</jk> MyResource(RestContext.Builder <jv>builder</jv>) <jk>throws</jk> Exception {
371 		 *
372 		 * 			<jc>// Using method on builder.</jc>
373 		 * 			<jv>builder</jv>.allowedMethodHeaders(<js>"PATCH"</js>);
374 		 * 		}
375 		 *
376 		 * 		<jc>// Option #3 - Defined via builder passed in through init method.</jc>
377 		 * 		<ja>@RestInit</ja>
378 		 * 		<jk>public void</jk> init(RestContext.Builder <jv>builder</jv>) <jk>throws</jk> Exception {
379 		 * 			<jv>builder</jv>.allowedMethodHeaders(<js>"PATCH"</js>);
380 		 * 		}
381 		 * 	}
382 		 * </p>
383 		 *
384 		 * <h5 class='section'>Notes:</h5><ul>
385 		 * 	<li class='note'>
386 		 * 		Method names are case-insensitive.
387 		 * 	<li class='note'>
388 		 * 		Use <js>"*"</js> to represent all methods.

752 		/**
753 		 * Child REST resources.
754 		 *
755 		 * <p>
756 		 * Defines children of this resource.
757 		 *
758 		 * <p>
759 		 * A REST child resource is simply another servlet or object that is initialized as part of the ascendant resource and has a
760 		 * servlet path directly under the ascendant resource object path.
761 		 * <br>The main advantage to defining servlets as REST children is that you do not need to define them in the
762 		 * <c>web.xml</c> file of the web application.
763 		 * <br>This can cut down on the number of entries that show up in the <c>web.xml</c> file if you are defining
764 		 * large numbers of servlets.
765 		 *
766 		 * <p>
767 		 * Child resources must specify a value for {@link Rest#path() @Rest(path)} that identifies the subpath of the child resource
768 		 * relative to the ascendant path UNLESS you use the {@link RestContext.Builder#child(String, Object)} method to register it.
769 		 *
770 		 * <p>
771 		 * Child resources can be nested arbitrarily deep using this technique (i.e. children can also have children).
772 		 *
773 		 * <dl>
774 		 * 	<dt>Servlet initialization:</dt>
775 		 * 	<dd>
776 		 * 		<p>
777 		 * 			A child resource will be initialized immediately after the ascendant servlet/resource is initialized.
778 		 * 			<br>The child resource receives the same servlet config as the ascendant servlet/resource.
779 		 * 			<br>This allows configuration information such as servlet initialization parameters to filter to child
780 		 * 			resources.
781 		 * 		</p>
782 		 * 	</dd>
783 		 * 	<dt>Runtime behavior:</dt>
784 		 * 	<dd>
785 		 * 		<p>
786 		 * 			As a rule, methods defined on the <c>HttpServletRequest</c> object will behave as if the child
787 		 * 			servlet were deployed as a top-level resource under the child's servlet path.
788 		 * 			<br>For example, the <c>getServletPath()</c> and <c>getPathInfo()</c> methods on the
789 		 * 			<c>HttpServletRequest</c> object will behave as if the child resource were deployed using the
790 		 * 			child's servlet path.
791 		 * 			<br>Therefore, the runtime behavior should be equivalent to deploying the child servlet in the
792 		 * 			<c>web.xml</c> file of the web application.
793 		 * 		</p>
794 		 * 	</dd>
795 		 * </dl>
796 		 *
797 		 * <h5 class='section'>Example:</h5>
798 		 * <p class='bjava'>
799 		 * 	<jc>// Our child resource.</jc>
800 		 * 	<ja>@Rest</ja>(path=<js>"/child"</js>)
801 		 * 	<jk>public class</jk> MyChildResource {...}
802 		 *
803 		 * 	<jc>// Option #1 - Registered via annotation.</jc>
804 		 * 	<ja>@Rest</ja>(children={MyChildResource.<jk>class</jk>})
805 		 * 	<jk>public class</jk> MyResource {
806 		 *
807 		 * 		<jc>// Option #2 - Registered via builder passed in through resource constructor.</jc>
808 		 * 		<jk>public</jk> MyResource(RestContext.Builder <jv>builder</jv>) <jk>throws</jk> Exception {
809 		 *
810 		 * 			<jc>// Using method on builder.</jc>
811 		 * 			<jv>builder</jv>.children(MyChildResource.<jk>class</jk>);
812 		 *
813 		 * 			<jc>// Use a pre-instantiated object instead.</jc>
814 		 * 			<jv>builder</jv>.child(<js>"/child"</js>, <jk>new</jk> MyChildResource());
815 		 * 		}
816 		 *
817 		 * 		<jc>// Option #3 - Registered via builder passed in through init method.</jc>
818 		 * 		<ja>@RestInit</ja>
819 		 * 		<jk>public void</jk> init(RestContext.Builder <jv>builder</jv>) <jk>throws</jk> Exception {
820 		 * 			<jv>builder</jv>.children(MyChildResource.<jk>class</jk>);
821 		 * 		}
822 		 * 	}
823 		 * </p>
824 		 *
825 		 * <h5 class='section'>Notes:</h5><ul>
826 		 * 	<li class='note'>
827 		 * 		When defined as classes, instances are resolved using the registered bean store which
828 		 * 		by default is {@link BeanStore} which requires the class have one of the following
829 		 * 		constructors:
830 		 * 		<ul>
831 		 * 			<li><code><jk>public</jk> T(RestContext.Builder)</code>
832 		 * 			<li><code><jk>public</jk> T()</code>
833 		 * 		</ul>
834 		 * </ul>
835 		 *
836 		 * <h5 class='section'>See Also:</h5><ul>
837 		 * 	<li class='ja'>{@link Rest#children()}
838 		 * </ul>
839 		 *
840 		 * @param values
841 		 * 	The values to add to this setting.
842 		 * 	<br>Objects can be any of the specified types:
843 		 * 	<ul>
844 		 * 		<li>A class that has a constructor described above.
845 		 * 		<li>An instantiated resource object (such as a servlet object instantiated by a servlet container).
846 		 * 		<li>An instance of {@link RestChild} containing an instantiated resource object and a subpath.
847 		 * 	</ul>
848 		 * @return This object.
