23  /**
24   * Describes a single operation parameter.
25   *
26   * <p>
27   * A unique parameter is defined by a combination of a name and location.
28   *
29   * <p>
30   * There are five possible parameter types.
31   * <ul class='spaced-list'>
32   * 	<li><js>"path"</js> - Used together with Path Templating, where the parameter value is actually part of the
33   * 		operation's URL.
34   * 		This does not include the host or base path of the API.
35   * 		For example, in <code>/items/{itemId}</code>, the path parameter is <code>itemId</code>.
36   * 	<li><js>"query"</js> - Parameters that are appended to the URL.
37   * 		For example, in <code>/items?id=###</code>, the query parameter is <code>id</code>.
38   * 	<li><js>"header"</js> - Custom headers that are expected as part of the request.
39   * 	<li><js>"body"</js> - The payload that's appended to the HTTP request.
40   * 		Since there can only be one payload, there can only be one body parameter.
41   * 		The name of the body parameter has no effect on the parameter itself and is used for documentation purposes
42   * 		only.
43   * 		Since Form parameters are also in the payload, body and form parameters cannot exist together for the same
44   * 		operation.
45   * 	<li><js>"formData"</js> - Used to describe the payload of an HTTP request when either
46   * 		<code>application/x-www-form-urlencoded</code>, <code>multipart/form-data</code> or both are used as the
47   * 		content type of the request (in Swagger's definition, the consumes property of an operation).
48   * 		This is the only parameter type that can be used to send files, thus supporting the file type.
49   * 		Since form parameters are sent in the payload, they cannot be declared together with a body parameter for the
50   * 		same operation.
51   * 		Form parameters have a different format based on the content-type used (for further details, consult
52   * 		<code>http://www.w3.org/TR/html401/interact/forms.html#h-17.13.4</code>):
53   * 		<ul>
54   * 			<li><js>"application/x-www-form-urlencoded"</js> - Similar to the format of Query parameters but as a
55   * 				payload.
56   * 				For example, <code>foo=1&amp;bar=swagger</code> - both <code>foo</code> and <code>bar</code> are form
57   * 				parameters.
58   * 				This is normally used for simple parameters that are being transferred.
59   * 			<li><js>"multipart/form-data"</js> - each parameter takes a section in the payload with an internal header.
60   * 				For example, for the header <code>Content-Disposition: form-data; name="submit-name"</code> the name of
61   * 				the parameter is <code>submit-name</code>.
62   * 				This type of form parameters is more commonly used for file transfers.
63   * 		</ul>
64   * 	</li>
65   * </ul>
66   *
67   * <h5 class='section'>Example:</h5>
68   * <p class='bcode'>
69   * 	<jc>// Construct using SwaggerBuilder.</jc>
70   * 	ParameterInfo x = <jsm>parameterInfo</jsm>(<js>"query"</js>, <js>"foo"</js>);
71   *
72   * 	<jc>// Serialize using JsonSerializer.</jc>
73   * 	String json = JsonSerializer.<jsf>DEFAULT</jsf>.toString(x);
74   *
75   * 	<jc>// Or just use toString() which does the same as above.</jc>
76   * 	String json = x.toString();
77   * </p>
78   * <p class='bcode'>
79   * 	<jc>// Output</jc>
80   * 	{
81   * 		<js>"in"</js>: <js>"query"</js>,
82   * 		<js>"name"</js>: <js>"foo"</js>
83   * 	}
84   * </p>
85   */
