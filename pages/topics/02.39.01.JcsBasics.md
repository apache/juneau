Juneau supports converting arbitrary POJOs to canonical JSON per [RFC 8785](https://www.rfc-editor.org/rfc/rfc8785) (JCS - JSON Canonicalization Scheme) using a native serializer. JCS produces a deterministic, byte-for-byte representation suitable for cryptographic operations such as hashing and digital signing.

The <a href="/site/apidocs/org/apache/juneau/json/JcsSerializer.html" target="_blank">JcsSerializer</a> converts POJOs to canonical JSON. The <a href="/site/apidocs/org/apache/juneau/marshaller/Jcs.html" target="_blank">Jcs</a> marshaller pairs it with <a href="/site/apidocs/org/apache/juneau/json/JsonParser.html" target="_blank">JsonParser</a> for full round-trip support (JCS output is valid JSON).

##### Maven Dependency

JCS support is included in `juneau-marshall`:

```xml
<dependency>
    <groupId>org.apache.juneau</groupId>
    <artifactId>juneau-marshall</artifactId>
    <version>${juneau.version}</version>
</dependency>
```

##### Quick Start

```java
// A simple bean
public class Person {
    public String name;
    public int age;
    public Person() {}
    public Person(String name, int age) { this.name = name; this.age = age; }
}

// Serialize to canonical JSON
Person alice = new Person("Alice", 30);
String canonical = Jcs.of(alice);
```

Produces:

```json
{"age":30,"name":"Alice"}
```

##### Parsing

JCS output is valid JSON. Use the standard `JsonParser` or the `Jcs` marshaller:

```java
// Parse canonical JSON into a bean
Person parsed = Jcs.to(canonical, Person.class);

// Or use JsonParser directly
Person also = JsonParser.DEFAULT.parse(canonical, Person.class);
```

##### Canonicalization Rules (RFC 8785)

- **No whitespace** - Output is a single contiguous line.
- **Sorted properties** - Object keys sorted by UTF-16 code unit order, recursively.
- **ECMAScript numbers** - Shortest representation, no trailing zeros, lowercase `e`, `+` for positive exponents.
- **Strings** - Control chars escaped; non-ASCII emitted as literal UTF-8.
- **Array order** - Preserved (not sorted).

##### Bean-to-JCS Mapping

- **Maps and beans** → Canonical JSON objects (keys sorted)
- **Collections and arrays** → JSON arrays (order preserved)
- **Numbers** → ECMAScript-compatible strings
- **Strings** → JSON strings with JCS escaping
- **Booleans** → `true` / `false`
- **Null** → `null`
- **ObjectSwaps** → Applied for dates, enums, etc.

##### Media Types

- **Produces**: `application/json` (canonical form)
- **Accepts**: Standard JSON (use `JsonParser`)

##### REST Integration

```java
// JCS is included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }
```

##### Limitations

- **NaN and Infinity** - `Double.NaN`, `Double.POSITIVE_INFINITY`, and `Double.NEGATIVE_INFINITY` are rejected and throw `SerializeException`.
- **Lone surrogates** - Strings containing unpaired UTF-16 surrogate code units throw `SerializeException`.
- **BigDecimal/BigInteger** - Values beyond IEEE 754 double precision may lose precision or throw.
- **No separate media type** - Output is `application/json`. Use JCS explicitly by class reference, not content negotiation.

##### Use Cases

- **Digital signatures** - Canonical form ensures identical bytes for signers and verifiers.
- **Content hashing** - Deterministic output for SHA-256 or other hashes.
- **JWK Thumbprint** - Compatible with RFC 7638 and similar schemes.
- **Stable serialization** - Same object always produces the same string.
