The <a href="/site/apidocs/org/apache/juneau/yaml/annotation/YamlConfig.html" target="_blank">@YamlConfig</a> annotation is used to apply
YAML serializer and parser configuration annotations to REST resources, methods, and other contexts.

##### REST Integration

YAML support can be added to REST resources using the `BasicYamlConfig` interface or by including the YAML serializer
and parser in `BasicUniversalConfig`:

```java
// YAML-only REST resource
@Rest
public class MyYamlResource extends RestServlet implements BasicYamlConfig {

    @RestGet("/person/{id}")
    public Person getPerson(@Path int id) {
        return findPerson(id);
    }
}

// Universal resource supporting JSON, XML, HTML, UON, and YAML
@Rest
public class MyUniversalResource extends RestServlet implements BasicUniversalConfig {

    @RestGet("/person/{id}")
    public Person getPerson(@Path int id) {
        return findPerson(id);
    }
}
```

##### Custom Configuration

You can configure the YAML serializer and parser through annotation or programmatic builders:

```java
@Rest(
    serializers = YamlSerializer.class,
    parsers = YamlParser.class,
    defaultAccept = "application/yaml"
)
public class MyResource extends RestServlet { ... }
```
