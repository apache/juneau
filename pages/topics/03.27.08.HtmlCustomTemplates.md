When <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlDocSerializer.html" target="_blank">HtmlDocSerializer</a> wraps a serialized POJO inside a complete HTML document, the layout of that document is controlled by a <em>page template</em>.
By default it uses <a href="/site/apidocs/org/apache/juneau/marshall/html/BasicHtmlDocTemplate.html" target="_blank">BasicHtmlDocTemplate</a> (described in [BasicHtmlDocTemplate](/docs/topics/BasicHtmlDocTemplate)), but you can supply your own to take complete control over the generated markup.

Custom page templates are created by implementing the <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlDocTemplate.html" target="_blank">HtmlDocTemplate</a> interface and associating it with your serializer using the <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlDocSerializer.Builder.html#template(java.lang.Class)" target="_blank">HtmlDocSerializer.Builder.template(Class)</a> setting.

The interface defines a single `writeTo(session, writer, pojo)` method, and the implementation is open-ended &mdash; you can write out the page contents any way you wish using the <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlWriter.html" target="_blank">HtmlWriter</a> passed to it.
The serialized POJO is available from the session (typically written into the body via `session.parentSerialize(writer, pojo)`).

## Implementing a template from scratch

The following template ignores the header/nav/aside/footer settings entirely and renders a bare-bones document consisting of just the serialized POJO wrapped in a `<body>`:

```java
public class MyTemplate implements HtmlDocTemplate {

    @Override
    public void writeTo(HtmlDocSerializerSession session, HtmlWriter w, Object o) throws Exception {
        w.sTag("html").nl(0);
        w.sTag(1, "body").nl(1);
        // Serialize the POJO into the body using the parent HtmlSerializer.
        session.parentSerialize(w, o);
        w.eTag(1, "body").nl(1);
        w.eTag("html").nl(0);
    }
}
```

Wire it into a serializer using the builder:

```java
HtmlDocSerializer serializer = HtmlDocSerializer
    .create()
    .template(MyTemplate.class)
    .build();
```

## Extending the basic template

Implementing `HtmlDocTemplate` from scratch means reproducing the entire page structure.
In most cases it's easier to subclass <a href="/site/apidocs/org/apache/juneau/marshall/html/BasicHtmlDocTemplate.html" target="_blank">BasicHtmlDocTemplate</a> and override only the individual section methods you want to change.
`BasicHtmlDocTemplate` breaks the page down into overridable `protected` methods, each receiving the session, writer, and POJO:

- `head(session, w, o)` &mdash; the `<head>` element (delegates to `style(...)` and `script(...)`).
- `header(session, w, o)` &mdash; the `<header>` element (page title/description/branding).
- `nav(session, w, o)` &mdash; the `<nav>` element (navigation links).
- `aside(session, w, o)` &mdash; the `<aside>` element (side-bar text).
- `article(session, w, o)` &mdash; the `<article>` element (the serialized POJO itself).
- `footer(session, w, o)` &mdash; the `<footer>` element (footer message).

There are also companion `hasHeader(...)`, `hasNav(...)`, `hasAside(...)`, `hasFooter(...)`, `hasStyle(...)`, and `hasScript(...)` predicates that control whether each section is rendered.

The following example overrides just the footer to append a fixed copyright notice while leaving every other section untouched:

```java
public class MyTemplate extends BasicHtmlDocTemplate {

    @Override
    protected void footer(HtmlDocSerializerSession session, HtmlWriter w, Object o) throws Exception {
        // Preserve the default footer behavior...
        super.footer(session, w, o);
        // ...then add our own content.
        w.appendln(3, "<p>Copyright (c) My Company</p>");
    }
}
```
