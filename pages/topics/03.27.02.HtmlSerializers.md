The <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlSerializer.html" target="_blank">HtmlSerializer</a> class is used to serialize POJOs into
HTML.
The <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlDocSerializer.html" target="_blank">HtmlDocSerializer</a> class is the same but wraps the
serialized POJO inside a document template consisting of header, nav, aside, and footer sections.

## Data type representations

The following tables are the complete reference for how each POJO datatype is rendered by `HtmlSerializer`.
They mirror how the same data structures are represented in JSON.
See [HTML Methodology](/docs/topics/HtmlMethodology) for a conceptual overview of the mapping rules.

### Maps

`Maps` and beans are represented as tables.
The `_type` attribute is added to differentiate between objects (`Maps`/beans) and arrays (arrays/`Collections`).

<table class="code-table">
<tr>
<th>Data type</th>
<th>JSON example</th>
<th>HTML</th>
</tr>
<tr>
<td>Map</td>
<td>

```json
{
  k1: 'v1',
  k2: null
}
```

</td>
<td>

```html
<table _type='object'>
  <tr>
    <td>k1</td>
    <td>v1</td>
  </tr>
  <tr>
    <td>k2</td>
    <td><null></null></td>
  </tr>
</table>
```

</td>
</tr>
<tr>
<td>Map</td>
<td>

```json
{
  k1: 123,
  k2: 1.23,
  k3: null
}
```

</td>
<td>

```html
<table _type='object'>
  <tr>
    <td>k1</td>
    <td><number>123</number></td>
  </tr>
  <tr>
    <td>k2</td>
    <td><number>1.23</number></td>
  </tr>
  <tr>
    <td>k3</td>
    <td><null></null></td>
  </tr>
</table>
```

</td>
</tr>
<tr>
<td>Map</td>
<td>

```json
{
  k1: 'v1',

  k3: 1.23,

  k5: null
}
```

</td>
<td>

```html
<table _type='object'>
  <tr>
    <td>k1</td>
    <td>v1</td>
  </tr>
  <tr>
    <td>k2</td>
    <td><number>123</number></td>
  </tr>
  <tr>
    <td>k3</td>
    <td><number>1.23</number></td>
  </tr>
  <tr>
    <td>k4</td>
    <td><boolean>true</boolean></td>
  </tr>
  <tr>
```
