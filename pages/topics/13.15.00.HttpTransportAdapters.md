## Overview

The canonical `RestClient` in [org.apache.juneau.rest.client](/site/apidocs/org/apache/juneau/rest/client/package-summary.html) is transport-agnostic. Every HTTP
call is routed through an `HttpTransport` SPI, which is responsible for executing a fully-resolved
`TransportRequest` and returning a `TransportResponse`.

```
RestClient  ŌåÆ  HttpTransport  ŌåÆ  (Apache HC 4.5 / Apache HC 5 / OkHttp / Jetty / JDK / Mock)
```

`RestClient`'s POJO marshalling, remote proxies, interceptors, error-code handling, and logging are
all independent of which transport is in use. Swapping transports requires one line of configuration.

---

## Default Transport

The built-in `JavaHttpTransport` ŌĆö backed by the JDK's `java.net.http.HttpClient` ŌĆö is the
default. It is part of the `juneau-rest-client` artifact and requires no additional dependencies
on Java 11+. For most applications it is the right starting point.

```java
import org.apache.juneau.rest.client.*;

// No transport module needed ŌĆö JavaHttpTransport is picked up automatically.
RestClient client = RestClient.create()
    .marshaller(Json5.DEFAULT)
    .rootUrl("https://api.example.com")
    .build();
```

## Transport Discovery

`RestClient` discovers the transport to use in the following order of precedence:

1. **Explicit call** ŌĆö `RestClient.Builder.transport(HttpTransport)` always wins.
2. **`ServiceLoader` auto-discovery** ŌĆö each optional transport module registers an
   `HttpTransportProvider` via `ServiceLoader`. The highest-priority provider on the classpath is
   selected automatically when you add the module's Maven dependency.
3. **Built-in fallback** ŌĆö `JavaHttpTransport` is used when nothing else is found.

To set a transport explicitly:

```java
import org.apache.juneau.rest.client.*;
import org.apache.juneau.rest.client.apachehttpclient45.*;

RestClient client = RestClient.create()
    .transport(ApacheHc45Transport.create())
    .build();
```

## The HttpTransport SPI

`HttpTransport` is a single-method interface:

```java
public interface HttpTransport extends Closeable {
    TransportResponse execute(TransportRequest request) throws TransportException;
}
```

`TransportRequest` carries the fully-resolved method, URI, headers, and optional body.
`TransportResponse` carries the status code, reason phrase, response headers, and the raw
`InputStream` body (or nothing for status-only responses). Juneau's marshalling layer converts
between POJO types and these low-level streams.

Custom transports can be written by implementing this interface directly ŌĆö useful for testing,
protocol bridges, or non-HTTP backends:

```java
HttpTransport myTransport = request -> TransportResponse.builder()
    .statusCode(200)
    .reasonPhrase("OK")
    .body(new ByteArrayInputStream("{}".getBytes()))
    .build();

RestClient client = RestClient.create()
    .transport(myTransport)
    .build();
```

## Resource Lifecycle

`RestRequest` is the only `Closeable` in the user-facing API. Always wrap calls in
try-with-resources to release the underlying connection promptly:

```java
try (var req = client.get("/users/123")) {
    var user = req.run().as(User.class);
}

// At application shutdown
client.shutdown();
```

`client.shutdown()` delegates to `HttpTransport.close()`, which releases connection pools, threads,
and other resources held by the native HTTP client.
