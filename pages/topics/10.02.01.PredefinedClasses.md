The following example represents the bare-minimum needed for deploying a top-level REST endpoint with basic JSON
marshalling support:

```java
@Rest(
    path="/mypath",
    serializers=JsonSerializer.class,
    parsers=JsonParser.class
)
public class MyResource extends RestServlet {

    @RestGet(path="/")
    public Object getPojo() {
        ...
    }
}
```

The <a href="/site/apidocs/org/apache/juneau/rest/server/servlet/RestServlet.html" target="_blank">RestServlet</a> class provides all the logic for
starting up your REST application when the servlet container calls <a href="/site/apidocs/org/apache/juneau/rest/server/servlet/RestServlet.html#<init>()" target="_blank">init(ServletConfig)</a>.
On startup, it scans your class for annotations and sets up all of your serializers and parsers.
It then does this recursively for all child resources.

Users will typically not extend directly from <a href="/site/apidocs/org/apache/juneau/rest/server/servlet/RestServlet.html" target="_blank">RestServlet</a>.
Instead, several classes are provided by the framework to provide additional functionality and to handle different
use-cases.

Users will typically extend from one of the `Basic*` classes in one of three families: the **servlet family**,
the **resource family**, and the **Spring Boot family**. Cross-cutting endpoints (swagger docs, static files,
stats, error, favicon, navigation) are supplied by single-responsibility **mixin classes** rather than by
interfaces.

## Servlet family

The servlet family extends `HttpServlet` and is the standard choice for top-level resources deployed in a
servlet container (standalone Jetty, embedded Tomcat, Spring Boot via `ServletRegistrationBean`).

## Mixin family

Cross-cutting endpoints are contributed by single-responsibility mixin classes in
[org.apache.juneau.rest.server.ops](/site/apidocs/org/apache/juneau/rest/server/ops/package-summary.html). Mixins are composed into a host via `@Rest(mixins=...)`.
`BasicRestServlet` composes all of them by default; you can opt out via `@Rest(noInherit={"mixins"})` and
re-declare only what you need.

The `Basic*` classes compose the op-mixin endpoints via `@Rest(mixins=...)`.
`BasicRestServlet` declares:

```java
@Rest(mixins={SwaggerUiMixin.class, RedocMixin.class, ErrorMixin.class, HtdocMixin.class, StatsMixin.class, FaviconMixin.class})
public abstract class BasicRestServlet extends RestServlet implements BasicUniversalConfig { ... }
```

`BasicRestServletGroup` adds `NavigationMixin` on top:

```java
@Rest(mixins={NavigationMixin.class})
public abstract class BasicRestServletGroup extends BasicRestServlet { ... }
```

The `Basic*` classes also implement <a href="/site/apidocs/org/apache/juneau/rest/server/config/BasicJsonConfig.html" target="_blank">BasicJsonConfig</a>
(or `BasicUniversalConfig` on `BasicRestServlet`) which provides marshalling support.
Other config interfaces are available as well to quickly provide different types of marshalling support.
Note that these interfaces do not define any methods but rather simply provide a set of commonly-used annotations so
that you don't need to define them on all your classes.

For example, if you want to provide a resource that supports all languages in Juneau, simply add the <a href="/site/apidocs/org/apache/juneau/rest/server/config/BasicUniversalConfig.html" target="_blank">BasicUniversalConfig</a> interface like so:

```java
@Rest(...)
public class MyResource extends RestServlet implements BasicUniversalConfig {
    ...
}
```

The `*Spring*` classes are meant to be used in Spring Boot environments so that you can take full advantage of the
Spring Framework for injecting dependencies on child resources and helper classes.

The resource family (`BasicRestResource`, `BasicRestResourceGroup`) provides the same functionality as the
servlet classes but does not extend from <a href="https://jakarta.ee/specifications/servlet/6.0/apidocs/jakarta/servlet/http/HttpServlet.html" target="_blank">HttpServlet</a>.
This is important in Spring Boot environments where you may want to define child resources as Spring beans but
don't want Spring Boot to auto-detect them as servlets to be deployed as top-level resources.

## Choosing the right base class

| Use case | Base class | Notes |
|---|---|---|
| Single REST resource (non-Spring) | `BasicRestServlet` or `BasicRestResource` | Use when you don't need Spring bean injection |
| Single REST resource (Spring Boot) | `BasicSpringRestServlet` | Enables `@Autowired` in child resources |
| Router page (non-Spring) | `BasicRestServletGroup` | Mounts child resources |
| Router page (Spring Boot) | `BasicSpringRestServletGroup` | Mounts child resources with Spring bean resolution |
