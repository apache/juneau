The URL-Encoding format provides two annotations for tailoring how beans are serialized and parsed:
<a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncoding.html" target="_blank">@UrlEncoding</a>
for per-bean overrides, and
<a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncodingConfig.html" target="_blank">@UrlEncodingConfig</a>
for configuring REST classes and methods.

## `@UrlEncoding`

The <a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncoding.html" target="_blank">@UrlEncoding</a>
annotation can be applied to classes, fields, and methods to override the behavior of
<a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncodingSerializer.html" target="_blank">UrlEncodingSerializer</a>
and <a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncodingParser.html" target="_blank">UrlEncodingParser</a>
for individual beans.

<tree>
<node-0><java-annotation><a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncoding.html" target="_blank">UrlEncoding</a></java-annotation></node-0>
<node-1><java-method-annotation><a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncoding.html#expandedParams()" target="_blank">expandedParams</a></java-method-annotation></node-1>
</tree>

The <a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncoding.html#expandedParams()" target="_blank">expandedParams</a>
setting forces bean properties of type array or `Collection` to be expanded into multiple key/value
pairings rather than a single nested value. This is equivalent to the serializer/parser
`expandedParams()` builder settings, but applies only to instances of the annotated bean.

```java
// Expand array/collection properties into repeated key=value pairs for this bean only.
@UrlEncoding(expandedParams=true)
public class MyBean {
    public int[] a = {1, 2, 3};
}

// Serializes to: a=1&a=2&a=3   (instead of a=@(1,2,3))
String urlencoded = UrlEncodingSerializer.DEFAULT.write(new MyBean());
```

## `@UrlEncodingConfig`

The <a href="/site/apidocs/org/apache/juneau/marshall/urlencoding/UrlEncodingConfig.html" target="_blank">@UrlEncodingConfig</a>
annotation applies the same configuration properties at the REST level — on `@Rest`-annotated classes
and `@RestOp`-annotated methods:

```java
@Rest(...)
@UrlEncodingConfig(expandedParams="true")
public class MyResource extends BasicRestServlet {
    ...
}
```

Its `expandedParams` attribute is a `String` (so it can carry an
<a href="/docs/topics/DefaultVarResolver">SVL variable</a> such as `"$C{myConfig}"`), and a `rank`
attribute can be used to control the order in which config annotations are applied.

:::note
When parsing multi-part parameters, prefer `Collection`/`List` bean property types over arrays —
arrays must be recreated from scratch each time a value is added.
:::
