The <a href="/site/apidocs/org/apache/juneau/marshall/Marshalled.html" target="_blank">@Marshalled</a> annotation is a
class-level annotation in `org.apache.juneau.marshall` that controls how a class is serialized and parsed across all
Juneau marshalling formats (JSON, XML, HTML, MessagePack, and the rest).

It is the **format-side** counterpart to
<a href="/site/apidocs/org/apache/juneau/commons/bean/BeanType.html" target="_blank">@BeanType</a>. `@Marshalled` carries the
marshalling-only attributes â€” marshalling strategy, type naming, bean dictionary, implementation class, interceptors,
POJO examples, and wire-format overrides â€” while the bean-modeling attributes (`properties`, `excludeProperties`,
`findFluentSetters`, `interfaceClass`, `stopClass`, `propertyNamer`, `unsorted`, ...) live on `@BeanType` in
`juneau-commons`.

```java
@Marshalled(typeName="address")
public class Address {
    public String street;
    public String city;
}
```

:::note Migration
Prior to the 10.0 refactor, the bean-modeling members now on `@BeanType` were part of `@Marshalled`. `@Marshalled`
retains only the marshalling-layer attributes documented below; see
[@BeanType Annotation](/docs/topics/BeanTypeAnnotation) for the model side.
:::

## Serialize / Parse Round-Trip

`@Marshalled` participates transparently in every marshaller. Here the `example` attribute supplies a canonical POJO
example, and the type round-trips through JSON5:

```java
@Marshalled(example="{street:'123 Main St',city:'Springfield'}")
public class Address {
    public String street;
    public String city;
}

Address a = new Address();
a.street = "123 Main St";
a.city = "Springfield";

// Serialize.
String json = Json5.DEFAULT.of(a);           // {street:'123 Main St',city:'Springfield'}

// Parse back.
Address a2 = Json5.DEFAULT.to(json, Address.class);
```

## Marshalling Strategy: `as`

The <a href="/site/apidocs/org/apache/juneau/marshall/Marshalled.html#as()" target="_blank">@Marshalled(as)</a> attribute
selects the marshalling strategy from the
<a href="/site/apidocs/org/apache/juneau/marshall/MarshalledAs.html" target="_blank">MarshalledAs</a> enum. The default is
`MarshalledAs.DETECT` (auto-detection). Use `MarshalledAs.STRING` to serialize via `toString()` and parse via a
`fromString`/`valueOf`/single-`String` constructor â€” the declarative replacement for the old `MarshallingStringSwap`
pattern.

```java
// Serialize/parse this type as a single string value.
@Marshalled(as=MarshalledAs.STRING)
public class PhoneNumber {
    private final String value;
    public PhoneNumber(String value) { this.value = value; }   // single-String constructor for parsing
    @Override public String toString() { return value; }        // used for serialization
}
```

See [@MarshalledAs Enum](/docs/topics/MarshalledAs) for full details.

## Type Naming: `typeName` / `typePropertyName`

`typeName` assigns a dictionary name to a class so the serializer can emit a type discriminator when the static type is
ambiguous (for example, a property declared as `Object`), and the parser can resolve the concrete class from that name.
`typePropertyName` overrides the property used to hold the discriminator (default `_type`).

```java
@Marshalled(typeName="cat")
public class Cat extends Animal {}

@Marshalled(typeName="dog", typePropertyName="kind")
public class Dog extends Animal {}
```

When a `Cat` is serialized through a property of the base `Animal` type, the output includes `_type:'cat'` so it can be
parsed back to the correct subclass. See [Bean Dictionaries](/docs/topics/BeanDictionaries) for how type names are
registered and resolved.

## Bean Dictionary: `dictionary`

The <a href="/site/apidocs/org/apache/juneau/marshall/Marshalled.html#dictionary()" target="_blank">@Marshalled(dictionary)</a>
attribute registers the set of classes that make up the bean dictionary for all properties in this class and its
subclasses. Combined with `typeName`, this lets polymorphic properties round-trip.

```java
@Marshalled(dictionary={Cat.class, Dog.class})
public class Pet {
    public Animal animal;   // resolved to Cat or Dog via _type on parse
}
```

The dictionary can also be registered globally on the marshalling context:

```java
WriterSerializer serializer = JsonSerializer
    .create()
    .beanDictionary(Cat.class, Dog.class)
    .build();
```

See [Bean Sub Types](/docs/topics/BeanSubTypes) for a worked subtype example.

## Implementation Class: `implClass`

For interfaces and abstract classes,
<a href="/site/apidocs/org/apache/juneau/marshall/Marshalled.html#implClass()" target="_blank">@Marshalled(implClass)</a>
specifies the concrete class to instantiate during parsing.

```java
@Marshalled(implClass=MyInterfaceImpl.class)
public interface MyInterface { ... }
```

## Bean Interceptor: `interceptor`

The <a href="/site/apidocs/org/apache/juneau/marshall/Marshalled.html#interceptor()" target="_blank">@Marshalled(interceptor)</a>
attribute registers a
<a href="/site/apidocs/org/apache/juneau/commons/bean/BeanInterceptor.html" target="_blank">BeanInterceptor</a> that can
transform property values in transit â€” rewriting values on read (serialize) and write (parse).

```java
// Redact a sensitive property on serialization.
public class AddressInterceptor extends BeanInterceptor<Address> {
    @Override
    public Object readProperty(Address bean, String name, Object value) {
        return "taxInfo".equals(name) ? "redacted" : value;
    }
}

@Marshalled(interceptor=AddressInterceptor.class)
public class Address {
    public String getTaxInfo() { ... }
    public void setTaxInfo(String value) { ... }
}
```

## POJO Example: `example`

The <a href="/site/apidocs/org/apache/juneau/marshall/Marshalled.html#example()" target="_blank">@Marshalled(example)</a>
attribute supplies an example instance of the class in JSON5 (Simplified JSON) format. Examples are surfaced in generated
Swagger/OpenAPI documents. The value supports
[VarResolver.DEFAULT](/docs/topics/DefaultVarResolver) substitution (e.g. `$C{myConfigVar}`).

```java
@Marshalled(example="{foo:'bar',baz:123}")
public class MyClass { ... }
```

The same example can be defined in code with the
<a href="/site/apidocs/org/apache/juneau/marshall/Example.html" target="_blank">@Example</a> annotation on a static field or
method â€” see [@Example Annotation](/docs/topics/ExampleAnnotation).

## Wire-Format Overrides

`@Marshalled` can override the default wire format for common datatypes on a per-class basis. Each override defaults to
`NOT_SET` (inherit the context default). The available attributes are:

| Attribute | Applies to | Enum |
|---|---|---|
| `calendarFormat` | `Calendar` | `CalendarFormat` |
| `dateFormat` | `Date` | `DateFormat` |
| `durationFormat` | `Duration` | `DurationFormat` |
| `temporalFormat` | `java.time.*` | `TemporalFormat` |
| `localeFormat` | `Locale` | `LocaleFormat` |
| `timeZoneFormat` | `TimeZone` | `TimeZoneFormat` |
| `binaryFormat` | `byte[]` | `BinaryFormat` |
| `enumFormat` | enums | `EnumFormat` |
| `uuidFormat` | `UUID` | `UuidFormat` |
| `bitSetFormat` | `BitSet` | `BitSetFormat` |
| `bigNumberFormat` | `BigInteger` / `BigDecimal` | `BigNumberFormat` |
| `booleanFormat` | `Boolean` / `boolean` | `BooleanFormat` |
| `floatFormat` | `Float` / `Double` (non-finite) | `FloatFormat` |
| `currencyFormat` | `Currency` | `CurrencyFormat` |
| `classFormat` | `Class` | `ClassFormat` |
| `periodFormat` | `Period` | `PeriodFormat` |

```java
// Serialize byte[] values as hex and enums as lower-hyphenated names for this class.
@Marshalled(binaryFormat=BinaryFormat.HEX, enumFormat=EnumFormat.LOWER_HYPHEN)
public class MyBean {
    public byte[] data;
    public MyEnum status;
}
```

## Applying to REST Classes and Methods

The same attributes can be applied in bulk to third-party classes without annotating them directly, using
<a href="/site/apidocs/org/apache/juneau/marshall/MarshalledConfig.html" target="_blank">@MarshalledConfig</a> (with
`on`/`onClass`) on `@Rest`-annotated classes and `@RestOp`-annotated methods via
<a href="/site/apidocs/org/apache/juneau/marshall/MarshalledApply.html" target="_blank">@MarshalledApply</a>.
