The [org.apache.juneau.commons.http](/site/apidocs/org/apache/juneau/commons/http/package-summary.html) package provides lightweight, dependency-free primitives for parsing and modeling HTTP header values — media types, content-negotiation ranges, and name/value parameters — as defined by RFC 2616.

## Media Types and Negotiation

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/MediaType.html" target="_blank">MediaType</a></java-class>
Describes a single media type used in content negotiation (e.g. `text/html;charset=UTF-8`). Provides constants for the common types Juneau supports.

```java
MediaType json = MediaType.JSON;   // application/json
```

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/MediaRange.html" target="_blank">MediaRange</a></java-class>
Describes a single range within an `Accept`-style header, including its quality (`q`) metric and extension parameters.

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/MediaRanges.html" target="_blank">MediaRanges</a></java-class>
A parsed `Accept` (or similar) header — a list of `MediaRange` values sorted most-acceptable first, with a `match(...)` method for selecting the best available media type.

```java
MediaRanges accept = MediaRanges.of("text/json;q=0.8, text/*;q=0.6, */*;q=0.0");
int best = accept.match(List.of(MediaType.JSON, MediaType.XML));
```

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/StringRange.html" target="_blank">StringRange</a></java-class>
Like `MediaRange` but for plain comma-delimited header values with an optional quality metric (e.g. `Accept-Encoding`).

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/StringRanges.html" target="_blank">StringRanges</a></java-class>
A parsed `Accept-Encoding` (or similar) header — a sorted list of `StringRange` values.

## Header Parsing

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/HeaderElement.html" target="_blank">HeaderElement</a></java-class>
Represents a parsed element from an HTTP header value — a name plus zero or more parameters (RFC 2616).

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/HeaderValueParser.html" target="_blank">HeaderValueParser</a></java-class>
A lightweight parser that splits comma-delimited header values into `HeaderElement` objects with semicolon-delimited parameters.

## Name/Value Parameters

### <java-interface><a href="/site/apidocs/org/apache/juneau/commons/http/NameValuePair.html" target="_blank">NameValuePair</a></java-interface>
A name/value pair parameter, typically used in HTTP headers and request parameters.

### <java-class><a href="/site/apidocs/org/apache/juneau/commons/http/BasicNameValuePair.html" target="_blank">BasicNameValuePair</a></java-class>
A simple immutable implementation of `NameValuePair`.
