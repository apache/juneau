The <a href="/site/apidocs/org/apache/juneau/marshall/xml/Xml.html#format()" target="_blank">@Xml(format)</a> annotation
tweaks how a POJO is rendered in XML.
Its value is one of the enum constants defined in <a href="/site/apidocs/org/apache/juneau/marshall/xml/XmlFormat.html" target="_blank">XmlFormat</a>,
and it can be applied to both bean classes and individual bean properties.

At the class level, a format changes the default treatment of every property on the bean (for example, rendering all
properties as attributes).
At the property level, a format changes the treatment of that one property (for example, promoting a single property to
an XML attribute or flattening a collection into the parent element).
Property-level formats override class-level defaults, so the two levels can be combined.

## In this section

The remaining <code>@Xml(format)</code> pages document each family with worked before/after examples:

- <a href="/docs/topics/XmlFormatAttrs">Attribute Formats</a> &mdash; `ATTR`, `ATTRS`, and `ELEMENT` for moving values between attributes and elements.
