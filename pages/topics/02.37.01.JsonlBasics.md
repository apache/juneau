Juneau supports converting arbitrary POJOs to and from JSONL (JSON Lines, also called NDJSON) using native serializers and parsers. JSONL is a text format where each line is a valid JSON value, separated by newlines. It is the standard format for LLM fine-tuning datasets, streaming AI inference, log aggregation, and bulk data pipelines.

The <a href="/site/apidocs/org/apache/juneau/jsonl/JsonlSerializer.html" target="_blank">JsonlSerializer</a> converts POJOs directly to JSONL format, and the <a href="/site/apidocs/org/apache/juneau/jsonl/JsonlParser.html" target="_blank">JsonlParser</a> creates POJOs directly from JSONL by parsing each line as a JSON value.

##### Maven Dependency

JSONL support is included in `juneau-marshall`:

```xml
<dependency>
    <groupId>org.apache.juneau</groupId>
    <artifactId>juneau-marshall</artifactId>
    <version>${juneau.version}</version>
</dependency>
```

##### Quick Start

```java
// A simple bean
public class Person {
    public String name;
    public int age;
    public Person() {}
    public Person(String name, int age) { this.name = name; this.age = age; }
}

// Serialize a list of beans to JSONL
List<Person> people = List.of(new Person("Alice", 30), new Person("Bob", 25));
String jsonl = Jsonl.of(people);
```

Produces:

```json
{"name":"Alice","age":30}
{"name":"Bob","age":25}
```

##### Parsing

```java
// Parse JSONL into a list of beans
List<Person> parsed = Jsonl.to(jsonl, List.class, Person.class);

// Parse into an array
Person[] array = Jsonl.to(jsonl, Person[].class);

// Parse a single line
Person p = Jsonl.to("{\"name\":\"Alice\",\"age\":30}", Person.class);

// Parse into a list of maps
List<JsonMap> maps = Jsonl.to(jsonl, List.class, JsonMap.class);
```

##### Serialization

```java
// Use the default serializer
String jsonl = JsonlSerializer.DEFAULT.serialize(someObject);

// Use the marshaller for convenience
String jsonl = Jsonl.of(someObject);
```

##### Bean-to-JSONL Mapping

The conversion follows the <a href="https://jsonlines.org/" target="_blank">JSONL specification</a>:

- **Collections and arrays** → One JSON value per line (no wrapping brackets)
- **Single bean or map** → One JSON line
- **Strings** → JSON string literals, one per line
- **Numbers** → JSON number literals, one per line
- **Booleans** → `true` / `false`, one per line
- **Null** → `null`, one per line
- **Nested objects** → Stay compact on a single line per outer element
- **ObjectSwaps** → Applied for non-JSON-primitive types (e.g. byte arrays, dates)

##### Media Types

- **Produces**: `application/jsonl`
- **Accepts**: `application/jsonl`, `application/x-ndjson`, `text/jsonl`

##### REST Integration

```java
// Use BasicJsonlConfig for JSONL-only REST resources
@Rest
public class MyResource extends RestServlet implements BasicJsonlConfig { ... }

// JSONL is also included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }
```

##### Limitations

- **No pretty-printing**: Each JSON value must fit on a single line; `useWhitespace()` is always suppressed.
- **First-line-only for single-object targets**: When parsing JSONL to a single non-collection type, only the first non-empty line is parsed; trailing lines are ignored.
- **No multi-line JSON input**: Each JSON value must be on a single line; pretty-printed JSON will cause parse errors.
