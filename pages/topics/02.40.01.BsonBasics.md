Juneau supports converting arbitrary POJOs to and from BSON (Binary JSON) using native serializers and parsers. BSON is the binary serialization format used by MongoDB, extending JSON with typed integers (int32/int64), native datetime, decimal128, and binary data. It uses little-endian byte order and length-prefixed documents.

The <a href="/site/apidocs/org/apache/juneau/bson/BsonSerializer.html" target="_blank">BsonSerializer</a> converts POJOs directly to BSON format, and the <a href="/site/apidocs/org/apache/juneau/bson/BsonParser.html" target="_blank">BsonParser</a> creates POJOs directly from BSON.

##### Maven Dependency

BSON support is included in `juneau-marshall`:

```xml
<dependency>
    <groupId>org.apache.juneau</groupId>
    <artifactId>juneau-marshall</artifactId>
    <version>${juneau.version}</version>
</dependency>
```

##### Quick Start

```java
// A simple bean
public class Person {
    public String name;
    public int age;
    public Person() {}
    public Person(String name, int age) { this.name = name; this.age = age; }
}

// Serialize a bean to BSON
Person alice = new Person("Alice", 30);
byte[] bson = Bson.of(alice);

// Parse BSON into a bean
Person parsed = Bson.to(bson, Person.class);
```

##### Parsing

```java
// Parse BSON into a bean
Person p = Bson.to(bson, Person.class);

// Parse into a map
JsonMap map = Bson.to(bson, JsonMap.class);

// Parse into a typed map
Map<String, Object> m = Bson.to(bson, Map.class, String.class, Object.class);

// Parse a list (root must be document; use wrapped {"value": [...]} for standalone arrays)
List<Person> people = Bson.to(bson, List.class, Person.class);
```

##### Serialization

```java
// Use the default serializer
byte[] bson = BsonSerializer.DEFAULT.serialize(someObject);

// Use the marshaller for convenience
byte[] bson = Bson.of(someObject);

// Custom serializer: write dates as ISO 8601 strings instead of BSON datetime
BsonSerializer s = BsonSerializer.create().writeDatesAsDatetime(false).build();
byte[] bson = s.serialize(someObject);
```

##### Java-to-BSON Type Mapping

| Java Type | BSON Type | Notes |
|-----------|-----------|-------|
| Bean, Map | Document (0x03) | Nested structures |
| Collection, array | Array (0x04) | Document with "0", "1", ... keys |
| String, Enum | String (0x02) | |
| int, short, byte | Int32 (0x10) | 4 bytes little-endian |
| long | Int64 (0x12) | 8 bytes little-endian |
| float, double | Double (0x01) | IEEE 754 |
| Date, Calendar, Instant | Datetime (0x09) | When writeDatesAsDatetime=true |
| Duration | String | ISO 8601 duration |
| byte[] | Binary (0x05) | Subtype 0x00 |
| BigDecimal, BigInteger | Decimal128 (0x13) | IEEE 754-2008 |
| null | Null (0x0A) | |

##### Media Types

- **Produces**: `application/bson`
- **Accepts**: `application/bson`

##### REST Integration

```java
// Use BasicBsonConfig for BSON-only REST resources
@Rest
public class MyResource extends RestServlet implements BasicBsonConfig { ... }

// BSON is also included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }

// RestClient with BSON
RestClient client = RestClient.create().uri("http://localhost:8080").bson().build();
byte[] response = client.get("/api/data").bson().run().body();
```

##### Limitations

- **Root must be a document**: BSON requires a top-level document. Standalone scalars, collections, and arrays are wrapped as `{value: x}`.
- **Map keys must be strings**: Non-string keys use `toString()`. Null keys use a configurable placeholder.
- **ObjectId is read-only**: BSON ObjectId (0x07) is parsed as a 24-character hex string; Juneau does not generate ObjectIds.
- **Deprecated types skipped**: Undefined, DBPointer, Symbol, JS with Scope are silently skipped during parsing.
- **MongoDB-internal types unsupported**: JavaScript, Timestamp, MinKey, MaxKey are not supported.
