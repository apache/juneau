Juneau supports converting arbitrary POJOs to and from the **Protobuf Text Format** using native serializers and parsers implemented without any external dependencies. This is the human-readable text format (not the binary wire format) used for protobuf configuration files, debugging, and interop with tools like `protoc --decode`. The <a href="/site/apidocs/org/apache/juneau/proto/ProtoSerializer.html" target="_blank">ProtoSerializer</a> converts POJOs directly to Protobuf Text Format, and the <a href="/site/apidocs/org/apache/juneau/proto/ProtoParser.html" target="_blank">ProtoParser</a> creates POJOs directly from protobuf text without intermediate DOM objects.

**No `.proto` schema files are required**â€”Java bean metadata provides the type information.

##### Maven Dependency

Protobuf Text Format support is included in `juneau-marshall`:

```xml
<dependency>
    <groupId>org.apache.juneau</groupId>
    <artifactId>juneau-marshall</artifactId>
    <version>${juneau.version}</version>
</dependency>
```

##### Quick Start

```java
// Serialize a bean to Protobuf Text Format
Person p = new Person("Alice", 30);
String proto = Proto.of(p);
```

Produces:

```
name: "Alice"
age: 30
```

##### Parsing

```java
// Parse Protobuf Text Format into a bean
Person parsed = Proto.to(proto, Person.class);

// Parse into a Map or JsonMap
JsonMap map = Proto.to(proto, JsonMap.class);
```

##### Serialization

```java
// Use the default serializer
String proto = ProtoSerializer.DEFAULT.serialize(someObject);

// Use the marshaller for convenience
String proto = Proto.of(someObject);
```

##### Bean-to-Protobuf Mapping

The conversion follows the <a href="https://protobuf.dev/reference/protobuf/textformat-spec" target="_blank">Protobuf Text Format specification</a>:

- **Maps and beans** â†’ Message fields with brace delimiters or colon syntax
- **Collections and arrays of simple values** â†’ List syntax `[v1, v2, ...]`
- **Collections of beans** â†’ Repeated field names
- **Strings** â†’ Quoted strings with C-style escaping
- **Numbers** â†’ Numeric literals (decimal, hex, octal)
- **Booleans** â†’ `true` / `false`
- **Enums** â†’ Unquoted identifiers
- **Date/time and Duration** â†’ ISO 8601 strings
- **null** â†’ Omitted (no null representation in protobuf text format)

##### Annotations

- **`@Proto`** - Add a comment above a field in the serialized output
- **`@ProtoConfig`** - Class-level: `useListSyntaxForBeans`, `useColonForMessages`

##### Media Types

- **Produces**: `text/protobuf`
- **Accept**: `text/protobuf`, `text/x-protobuf`

##### REST Integration

```java
// Use BasicProtoConfig for Protobuf-only REST resources
@Rest
public class MyResource extends RestServlet implements BasicProtoConfig { ... }

// Protobuf is also included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }
```
