Juneau supports converting arbitrary POJOs to and from HTML using ultra-efficient serializers and parsers.

The HTML support is built on top of the XML support.
The <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlSerializer.html" target="_blank">HtmlSerializer</a> converts POJOs directly to HTML without the need for intermediate DOM objects, and the <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlParser.html" target="_blank">HtmlParser</a> uses a STaX-based reader to reconstruct POJOs directly from that same markup.
Like the rest of Juneau, no annotations are required on your POJOs &mdash; beans and maps render as tables, collections and arrays render as bulleted lists, and simple values render as text with lightweight type tags where the datatype cannot be inferred through reflection.

The primary use case for HTML marshalling is rendering POJOs in an easy-to-read format in REST interfaces.
For that use case, the <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlDocSerializer.html" target="_blank">HtmlDocSerializer</a> wraps the serialized POJO inside a complete HTML document (header, nav, aside, article, and footer sections) so that a bean can be rendered as a self-contained, styled web page.

## Serializer and parser families

- <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlSerializer.html" target="_blank">HtmlSerializer</a> / <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlParser.html" target="_blank">HtmlParser</a> &mdash; convert POJOs to and from HTML fragments (tables and lists).
- <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlDocSerializer.html" target="_blank">HtmlDocSerializer</a> &mdash; extends `HtmlSerializer` to wrap the fragment inside a full HTML document driven by a customizable page template. Its output is also readable back in by `HtmlParser`.
- <a href="/site/apidocs/org/apache/juneau/marshall/html/HtmlSchemaSerializer.html" target="_blank">HtmlSchemaSerializer</a> &mdash; produces HTML renderings of the JSON-Schema documents that describe your serialized output.
