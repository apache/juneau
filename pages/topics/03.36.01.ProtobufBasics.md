Juneau supports converting arbitrary POJOs to and from the **Protobuf Text Format** using native serializers and parsers implemented without any external dependencies. This is the human-readable text format (not the binary wire format) used for protobuf configuration files, debugging, and interop with tools like `protoc --decode`. The <a href="/site/apidocs/org/apache/juneau/marshall/prototext/PrototextSerializer.html" target="_blank">PrototextSerializer</a> converts POJOs directly to Protobuf Text Format, and the <a href="/site/apidocs/org/apache/juneau/marshall/prototext/PrototextParser.html" target="_blank">PrototextParser</a> creates POJOs directly from protobuf text without intermediate DOM objects.

**No `.proto` schema files are required**â€”Java bean metadata provides the type information.

## Quick Start

```java
// Serialize a bean to Protobuf Text Format
Person p = new Person("Alice", 30);
String proto = Prototext.of(p);
```

Produces:

```
name: "Alice"
age: 30
```

## Parsing

```java
// Parse Protobuf Text Format into a bean
Person parsed = Prototext.to(proto, Person.class);

// Parse into a Map or JsonMap
JsonMap map = Prototext.to(proto, JsonMap.class);
```

## Serialization

```java
// Use the default serializer
String proto = PrototextSerializer.DEFAULT.serialize(someObject);

// Use the marshaller for convenience
String proto = Prototext.of(someObject);
```

## Bean-to-Protobuf Mapping

The conversion follows the <a href="https://protobuf.dev/reference/protobuf/textformat-spec" target="_blank">Protobuf Text Format specification</a>:

- **Maps and beans** â†’ Message fields with brace delimiters or colon syntax
- **Collections and arrays of simple values** â†’ List syntax `[v1, v2, ...]`
- **Collections of beans** â†’ Repeated field names
- **Strings** â†’ Quoted strings with C-style escaping
- **Numbers** â†’ Numeric literals (decimal, hex, octal)
- **Booleans** â†’ `true` / `false`
- **Enums** â†’ Unquoted identifiers
- **Date/time and Duration** â†’ ISO 8601 strings
- **null** â†’ Omitted (no null representation in protobuf text format)

## Annotations

- **`@Prototext`** - Add a comment above a field in the serialized output
- **`@PrototextConfig`** - Class-level: `useListSyntaxForBeans`, `useColonForMessages`

## Media Types

- **Produces**: `text/protobuf`
- **Accept**: `text/protobuf`, `text/x-protobuf`

## REST Integration

```java
// Use BasicPrototextConfig for Protobuf-only REST resources
@Rest
public class MyResource extends RestServlet implements BasicPrototextConfig { ... }

// Protobuf is also included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }
```
