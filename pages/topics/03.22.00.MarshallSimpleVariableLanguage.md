The <a href="/site/apidocs/org/apache/juneau/commons/svl/package-summary.html" target="_blank">org.apache.juneau.commons.svl</a>
package defines "Simple Variable Language" (SVL) — text containing variables of the form
`$varName{varKey}` that are resolved by a
<a href="/site/apidocs/org/apache/juneau/commons/svl/VarResolver.html" target="_blank">VarResolver</a>.
SVL is used extensively across the Config, REST, and Microservice APIs, and by many Juneau annotations
so that values can be sourced from configuration files, environment variables, and system properties.

The canonical reference for the language itself — the built-in source variables, the `#{...}`
function-call syntax and function catalog, the `VarTemplate` compiled-template API, and
`Supplier<String>` live-reload patterns — is
[Simple Variable Language](/docs/topics/SimpleVariableLanguage). This section collects the
marshall-context-specific SVL topics:

- [SVL Variables](/docs/topics/SvlVariables) — the variables available in a REST context.
- [VarResolvers](/docs/topics/VarResolvers) — creating and configuring resolvers.
- [VarResolver.DEFAULT](/docs/topics/DefaultVarResolver) — the reusable default resolver.
- [SVL Other Notes](/docs/topics/SvlOtherNotes) — escaping, recursion, and efficiency notes.

```java
// The default resolver handles $S (system property) and $E (environment) variables out of the box.
String property = VarResolver.DEFAULT.resolve("The Java home directory is $S{java.home}");
```
