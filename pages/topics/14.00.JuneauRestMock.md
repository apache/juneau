## Contents/Features

The `juneau-rest-mock` module contains convenience APIs for performing serverless unit testing of your REST APIs.
Each of the APIs provides the ability to fully test your server and client REST interfaces without the need for a
running servlet container.

Tests run entirely in-process.
The mock client performs full serialization and parsing of the HTTP request and response, then dispatches the request
directly to the resource's <a href="/site/apidocs/org/apache/juneau/rest/server/RestOpInvoker.html" target="_blank">RestOpInvoker</a>
rather than sending it over the wire.
This makes tests fast and self-contained while still exercising nearly all of the client- and server-side code paths —
the actual over-the-wire transmission is the only aspect being bypassed.

The module ships two flavors of mock client:

- The classic <a href="/site/apidocs/org/apache/juneau/rest/mock/classic/MockRestClient.html" target="_blank">MockRestClient</a> —
  built on the Apache HttpClient 4.5–based classic <a href="/site/apidocs/org/apache/juneau/rest/client/classic/RestClient.html" target="_blank">RestClient</a>,
  exposing its full fluent surface.
  This is the client documented in [Mock REST Client Overview](/docs/topics/MockRestClientOverview).
- The next-generation <a href="/site/apidocs/org/apache/juneau/rest/mock/MockRestClient.html" target="_blank">MockRestClient</a> —
  paired with the transport-agnostic <a href="/site/apidocs/org/apache/juneau/rest/client/RestClient.html" target="_blank">RestClient</a>,
  for tests that target the new stack.
  See [Next-Generation REST Client](/docs/topics/NextGenRestClient) for examples.

Both clients live in the same Maven module and can be mixed in a single project.
