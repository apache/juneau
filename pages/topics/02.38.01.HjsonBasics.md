Juneau supports converting arbitrary POJOs to and from Hjson (Human JSON) using native serializers and parsers. Hjson is a syntax extension of JSON designed for human-friendly configuration files and hand-edited data. It extends JSON with quoteless strings, multiline strings, comments, optional commas, and unquoted keys.

The <a href="/site/apidocs/org/apache/juneau/hjson/HjsonSerializer.html" target="_blank">HjsonSerializer</a> converts POJOs directly to Hjson format, and the <a href="/site/apidocs/org/apache/juneau/hjson/HjsonParser.html" target="_blank">HjsonParser</a> creates POJOs directly from Hjson.

##### Maven Dependency

Hjson support is included in `juneau-marshall`:

```xml
<dependency>
    <groupId>org.apache.juneau</groupId>
    <artifactId>juneau-marshall</artifactId>
    <version>${juneau.version}</version>
</dependency>
```

##### Quick Start

```java
// A simple bean
public class Person {
    public String name;
    public int age;
    public Person() {}
    public Person(String name, int age) { this.name = name; this.age = age; }
}

// Serialize to Hjson (readable mode)
Person alice = new Person("Alice", 30);
String hjson = Hjson.of(alice);
```

Produces:

```json
{
  name: Alice
  age: 30
}
```

##### Parsing

```java
// Parse Hjson into a bean
Person parsed = Hjson.to(hjson, Person.class);

// Parse into a map
JsonMap map = Hjson.to(hjson, JsonMap.class);

// Parse root-braceless Hjson (no outer {})
String rootBraceless = "name: Bob\nage: 25";
Person bob = Hjson.to(rootBraceless, Person.class);
```

##### Serialization Modes

```java
// Readable mode (default) - newlines between members, quoteless strings
String hjson = Hjson.DEFAULT.write(bean);

// Compact mode - single line, comma-separated
String compact = Hjson.DEFAULT_COMPACT.write(bean);
```

##### Hjson Syntax Highlights

- **Quoteless strings**: `name: Alice` (no quotes when unambiguous)
- **Multiline strings**: Triple single-quote blocks with indentation stripping
- **Comments**: `#`, `//`, and `/* ... */` block comments
- **Optional commas**: Newlines can separate object members
- **Trailing commas**: Ignored
- **Unquoted keys**: Object keys need not be quoted when safe

##### Bean-to-Hjson Mapping

- **Maps and beans** → Hjson objects
- **Collections and arrays** → Hjson arrays
- **Strings** → Quoteless (when safe), double-quoted, or multiline
- **Numbers** → Hjson numbers
- **Booleans** → `true` / `false`
- **Null** → `null`
- **ObjectSwaps** → Applied for non-primitive types (dates, enums, etc.)

##### Media Types

- **Produces**: `application/hjson`
- **Accepts**: `application/hjson`, `application/hjson+json`

##### REST Integration

```java
// Hjson is included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }
```

##### Round-Trip Notes

- Root-braceless Hjson can be parsed; serialization always emits root braces by default
- Strings equal to `true`, `false`, `null`, or parseable as numbers are automatically quoted on output to prevent ambiguity
- In compact mode, multiline strings use JSON escape sequences instead of `'''` blocks
