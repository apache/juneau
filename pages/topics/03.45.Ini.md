Juneau supports converting POJOs to and from the classic INI configuration file format using native serializers and parsers implemented without any external dependencies.
INI organizes data as sections (`[sectionName]`) containing key-value pairs, and is a familiar format for application and tool configuration files.
The <a href="/site/apidocs/org/apache/juneau/marshall/ini/IniSerializer.html" target="_blank">IniSerializer</a> converts POJOs directly to INI text, and the <a href="/site/apidocs/org/apache/juneau/marshall/ini/IniParser.html" target="_blank">IniParser</a> creates POJOs directly from INI text.

<DependencyInfo artifact="juneau-marshall" mavenOnly note="INI support is included in juneau-marshall." />

## Quick Start

The <a href="/site/apidocs/org/apache/juneau/marshall/marshaller/Ini.html" target="_blank">Ini</a> marshaller pairs a serializer and parser behind a single convenience API:

```java
public class Contact {
    public String name;
    public int age;
    public Address address;

    public Contact() {}
    public Contact(String name, int age, Address address) {
        this.name = name;
        this.age = age;
        this.address = address;
    }
}

public class Address {
    public String street;
    public String city;
    public String state;
    // constructors omitted
}

Contact contact = new Contact("Alice", 30, new Address("123 Main St", "Boston", "MA"));

// Serialize a bean to INI
String ini = Ini.DEFAULT.of(contact);

// Parse INI back into a bean
Contact parsed = Ini.DEFAULT.to(ini, Contact.class);
```

Produces:

```ini
name = Alice
age = 30

[address]
street = 123 Main St
city = Boston
state = MA
```

## Serialization

Use the default serializer directly, or the marshaller for convenience:

```java
// Use the default serializer instance (thread-safe and reusable)
String ini = IniSerializer.DEFAULT.serialize(myBean);

// DEFAULT_READABLE inserts blank lines between sections
String pretty = IniSerializer.DEFAULT_READABLE.serialize(myBean);

// Use the marshaller
String ini2 = Ini.DEFAULT.of(myBean);
```

### Bean-to-INI mapping

| Java construct | INI representation |
|---|---|
| Simple bean/map properties | Key-value pairs in the default (top) section |
| Nested beans | `[sectionName]` sections |
| Deeply nested beans | `[path/to/section]` section paths |
| `Map<String,?>` values | `[sectionName]` sections |
| Collections and complex map values | JSON5-encoded inline values |
| `null` values | Unquoted `null` token |

Collections and complex values are embedded as JSON5 inline strings so they remain fully round-trippable:

```ini
name = Alice
age = 30
tags = ['a','b','c']

[address]
street = 123 Main St
city = Boston
state = MA
```

:::note
The top-level object must be a bean or a `Map<String,?>`. Top-level collections, arrays, and scalar values are not supported and cause a `SerializeException`.
:::

## Parsing

```java
// Use the default parser instance
Contact contact = IniParser.DEFAULT.parse(ini, Contact.class);

// Parse into a generic map
Map<String,Object> map = IniParser.DEFAULT.parse(ini, Map.class, String.class, Object.class);

// Use the marshaller
Contact contact2 = Ini.DEFAULT.to(ini, Contact.class);
```

Values are interpreted by type:

| INI value | Parsed as |
|---|---|
| Unquoted `null` | Java `null` |
| Unquoted `true` / `false` | `Boolean` |
| Unquoted numbers | `Number` (int, long, float, or double as appropriate) |
| Single-quoted strings (`'...'`) | `String` (with `''` unescaped to a single quote) |
| Values starting with `[` or `{` | Delegated to the JSON parser (arrays / objects) |
| ISO 8601 date/time strings | `Date`, `Calendar`, or `java.time.*` when the target requires it |
| ISO 8601 duration strings | `java.time.Duration` |
| Other unquoted tokens | `String` |

Sections such as `[address]` map to nested bean or map properties, and section paths such as `[employment/company]` map to deeply nested properties.

## Configuration Options

The serializer builder exposes INI-specific formatting options:

```java
// Colon separator, no surrounding spaces, and property-description comments
IniSerializer s = IniSerializer.create()
    .kvSeparator(':')
    .spacedSeparator(false)
    .useComments()
    .build();

String ini = s.serialize(myBean);
```

| Builder method | Description | Default |
|---|---|---|
| `kvSeparator(char)` | Key-value separator character — `=` or `:` | `=` |
| `spacedSeparator(boolean)` | Add spaces around the separator (`key = value` vs `key=value`) | `true` |
| `useComments()` | Emit `#` comments from bean property descriptions | `false` |

The same options can be applied declaratively via the <a href="/site/apidocs/org/apache/juneau/marshall/ini/IniConfig.html" target="_blank">@IniConfig</a> annotation on classes and methods (`kvSeparator`, `spacedSeparator`, `useComments`, `rank`), which is the mechanism used to configure serializers and parsers on REST classes.

Per-property behavior can be customized with the <a href="/site/apidocs/org/apache/juneau/marshall/ini/Ini.html" target="_blank">@Ini</a> annotation on bean properties:

| Annotation attribute | Description |
|---|---|
| `section()` | Override the section header name for the property |
| `comment()` | Comment text emitted before the property |
| `json5Encoding()` | Force JSON5 encoding even for simple values |

## Media Types

- **Produces**: `text/ini`
- **Accepts**: `text/ini`, `text/x-ini`

## REST Integration

INI serializer and parser support are included in `BasicUniversalConfig`, so REST resources that implement it can produce and consume INI content alongside the other built-in formats:

```java
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }
```

Format-specific options can be applied to a resource with the `@IniConfig` annotation.
