The `juneau-test` module bundles a small JUnit 5 extension (formerly the standalone `juneau-junit5` module) for **test-time bean injection** — swapping production beans with test doubles inside a Juneau `BeanStore` for the duration of a JUnit 5 test, without modifying production code and without proxying. It is the Juneau-flavored answer to Spring's `@TestConfiguration` / `@MockBean`.

The extension lives in the <a href="/site/apidocs/org/apache/juneau/junit5/package-summary.html" target="_blank">org.apache.juneau.junit5</a> package and centers on three types:

- <a href="/site/apidocs/org/apache/juneau/junit5/JuneauBeanStoreExtension.html" target="_blank">JuneauBeanStoreExtension</a> — the JUnit 5 `Extension`, registered via `@ExtendWith` or `@RegisterExtension`.
- <a href="/site/apidocs/org/apache/juneau/junit5/TestBean.html" target="_blank">TestBean</a> — the field/method annotation that declares an override.
- <a href="/site/apidocs/org/apache/juneau/junit5/TestBeanStore.html" target="_blank">TestBeanStore</a> — the overlay bean store, resolvable as a test-method parameter.

## Quick start

```java
import static org.junit.jupiter.api.Assertions.*;

import org.apache.juneau.junit5.*;
import org.junit.jupiter.api.*;
import org.junit.jupiter.api.extension.*;

@ExtendWith(JuneauBeanStoreExtension.class)
class MyServiceTest {

    @TestBean
    ExternalApi mockApi = () -> "test-double";

    @Test
    void resolvesTheTestDouble(TestBeanStore store) {
        assertSame(mockApi, store.getBean(ExternalApi.class).orElseThrow());
    }

    interface ExternalApi { String describe(); }
}
```

## Two wiring modes

- **Mode INJECT** (default) — construction-time wiring. The system-under-test is built *after* the overlay, which is threaded into the builder's `overridingBeanStore(...)` setter. Universal: every bean type is eligible for replacement, including ones the framework would otherwise pin into a per-op memoizer at boot.
- **Mode OVERLAY** — push/pop overlays on a long-lived SUT (typically a `Microservice` booted once in `@BeforeAll`), applied and removed per test via `@TestBean(mode = Mode.OVERLAY)`.

The `overridingBeanStore(BeanStore)` setter is available on every builder implementing `BeanStoreOverridable<B>`: `MockRestClient.Builder`, `Microservice.Builder`, `SerializerSet.Builder`, `ParserSet.Builder`, and `EncoderSet.Builder`.
