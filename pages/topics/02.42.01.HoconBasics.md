Juneau supports converting arbitrary POJOs to and from HOCON (Human-Optimized Config Object Notation) using native serializers and parsers. HOCON is a superset of JSON used extensively in the JVM ecosystem (Akka, Play Framework, sbt, Apache Spark, Kafka, Flink) for configuration files. It extends JSON with path expressions, `=` as key-value separator, unquoted strings, object merging, substitutions (`${var}`), triple-quoted multi-line strings, and optional root braces.

The <a href="/site/apidocs/org/apache/juneau/hocon/HoconSerializer.html" target="_blank">HoconSerializer</a> converts POJOs to HOCON format, and the <a href="/site/apidocs/org/apache/juneau/hocon/HoconParser.html" target="_blank">HoconParser</a> creates POJOs from HOCON.

##### Maven Dependency

HOCON support is included in `juneau-marshall`:

```xml
<dependency>
    <groupId>org.apache.juneau</groupId>
    <artifactId>juneau-marshall</artifactId>
    <version>${juneau.version}</version>
</dependency>
```

##### Quick Start

```java
// A simple bean
public class Person {
    public String name;
    public int age;
    public Person() {}
    public Person(String name, int age) { this.name = name; this.age = age; }
}

// Serialize to HOCON
Person alice = new Person("Alice", 30);
String hocon = Hocon.of(alice);
```

Produces:

```hocon
name = Alice
age = 30
```

##### Parsing

```java
// Parse HOCON into a bean
Person parsed = Hocon.to(hocon, Person.class);

// Parse into a map
JsonMap map = Hocon.to(hocon, JsonMap.class);

// Parse with path expressions
String pathExpr = "database.host = localhost\ndatabase.port = 5432";
JsonMap config = Hocon.to(pathExpr, JsonMap.class);
```

##### Serialization Modes

```java
// Readable mode (default) - = separator, unquoted strings, root braces omitted
String hocon = Hocon.DEFAULT.write(bean);

// With root braces
String braced = HoconSerializer.DEFAULT_BRACES.serialize(bean);

// Compact mode - single line
String compact = HoconSerializer.DEFAULT_COMPACT.serialize(bean);
```

##### HOCON Syntax Highlights

- **Key-value separators**: `=` (convention), `:` (JSON-style), or space-brace for objects
- **Path expressions**: `a.b.c = value` expands to nested objects
- **Unquoted strings**: Keys and simple values need not be quoted
- **Object merging**: Duplicate keys merge instead of overwrite
- **`+=` array append**: `list += item` appends to an array or creates `[item]` if the key does not exist
- **Substitutions**: `${var}` and `${?var}` (optional) resolved after parse
- **Self-referential substitution**: `path = ${path}"/bin"` appends to the previous value (e.g. `/usr` → `/usr/bin`)
- **Value concatenation**: Adjacent values concatenate: `path = /usr "/local"` → `"/usr/local"`
- **Array concatenation**: `[1, 2] [3, 4]` → `[1, 2, 3, 4]`
- **Object concatenation**: `{a: 1} {b: 2}` → `{a: 1, b: 2}`
- **Triple-quoted strings**: `"""..."""` for multi-line
- **Comments**: `#` and `//` single-line
- **Optional root braces**: Top-level `{}` can be omitted

##### Bean-to-HOCON Mapping

- **Maps and beans** → HOCON objects
- **Collections and arrays** → HOCON arrays
- **Strings** → Unquoted (when safe), quoted, or triple-quoted
- **Numbers** → HOCON numbers
- **Booleans** → `true` / `false`
- **Null** → `null`
- **ObjectSwaps** → Applied for non-primitive types (dates, enums, etc.)

##### Media Types

- **Produces**: `application/hocon`
- **Accepts**: `application/hocon`

##### REST Integration

```java
// HOCON is included in BasicUniversalConfig
@Rest
public class MyResource extends RestServlet implements BasicUniversalConfig { ... }

// HOCON-only resource
@Rest
public class HoconResource extends RestServlet implements BasicHoconConfig { ... }
```

##### Round-Trip Notes

- Path expressions are parsed; serialization always uses nested-brace form
- Substitutions (`${var}`) are resolved during parse; the serializer writes concrete values
- Strings equal to `true`, `false`, `null`, or parseable as numbers are quoted on output to prevent ambiguity
